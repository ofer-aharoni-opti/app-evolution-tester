# Must-not-break invariants

## Pipeline order

`TenantContextMiddleware` **must** run after `UseRouting` so `httpContext.GetEndpoint()` resolves and `[AllowWithNoTenantHeader]` / `[RequireUserNameHeader]` are visible. Moving it earlier silently bypasses the opt-out attribute and rejects every public endpoint.

`RequestLoggingMiddleware` **must** be the outermost middleware so it captures the final status code and elapsed time, including responses generated by other middleware (e.g. tenant validation 400s). Reordering it inward will under-report failures.

## Domain encapsulation

- `ZubiModel`/`ZabaModel` `Hydrate` is **only** for repositories restoring persisted state. Calling it from a handler skips the validation and timestamping that `Create`/`UpdateDetails` enforce.
- All mutating state on domain models goes through public methods (`MarkAsProcessed`, `UpdateDetails`). Property setters are `private`/`init`. Bypassing this (reflection, EF tracking, etc.) breaks the "domain enforces its own invariants" guarantee.
- Infrastructure mapping must remain **manual** (no Mapster in Infrastructure). Mapster bypasses constructors and would mutate models without going through the domain API.

## Tenant context lifecycle

- `ITenantApplicationContext` is **scoped** and is initialized exactly once per request by `TenantContextMiddleware`. Code further down the stack must read `Context`, not the headers — the middleware has already validated and parsed.
- `TenantContextHandler` only adds tenant headers when they are absent on the outbound request. Caller-set headers are authoritative; do not change this without auditing all `HttpClient` usages that already set headers.

## Repository semantics

- `UpdateAsync` returns `false` when the row does not exist. This is **not** an upsert — controllers rely on the boolean to return `404`. Don't "improve" it by inserting on missing.
- Repositories are registered as **singletons**. The in-memory `ConcurrentDictionary` is shared across requests; replacing it with a non-thread-safe collection or making the repository scoped will lose data between requests.

## API versioning

- `AssumeDefaultVersionWhenUnspecified = true` and `DefaultApiVersion = 1.0`. Any new controller must explicitly declare its supported versions (`[ApiVersion("1.0")]`, etc.) — silent default-to-1.0 will surprise consumers if 1.0 is later deprecated.
- `CreateTestResponseV2` is the documented breaking-change pattern. New breaking changes must follow the same shape (new contract + `[MapToApiVersion]`), not mutate the existing contract.

## Authentication / authorization

There is **no authentication or authorization** in the template today. The tenant header is a routing/scoping signal, not a security boundary. Any auth scheme (JWT, API keys, etc.) added later must run **before** `TenantContextMiddleware` so unauthenticated requests are rejected before tenant resolution. Do not assume the current state is hardened.

## CORS

`AllowCredentials()` is set on the policy. Per browser rules this means `Access-Control-Allow-Origin: *` will not be returned — `SetIsOriginAllowed` echoes the specific origin instead. Do not switch to `AllowAnyOrigin()`; it is incompatible with `AllowCredentials()` and will silently break browser CORS.

`DevAllowedOrigins` is only honored when `Environment.IsDevelopment()` **or** `ALLOW_CORS=true`. Production deployments must not set `ALLOW_CORS=true`.
